什么是操作系统

作为扩展机器的操作系统

操作系统是向应用程序提供基本抽象的概念。（自顶向下的观点）

作为资源管理器的操作系统

操作系统用来管理一个复杂系统的各个部分。操作系统的任务是在相互竞争的程序之间有序地控制对处理器、存储器以及其他I/O接口设备的分配。

操作系统的历史

计算机硬件介绍

多线程不提供真正的并行处理。在一个时刻只有一个进程在运行，但是线程的切换时间则减少到纳秒数量级。

存储器系统采用一种分层次的结构。

存储器

存储器系统的顶层是CPU中的寄存器，采用与CPU相同的材料制成的，所以和CPU一样快，访问它们是没有时延的。

下一层是高速缓存，多数由硬件控制。主存被分割成高速缓存行（cache line），最常用的高速缓存行放置在CPU内部或者非常接近CPU的高速缓存中。

再往下一层是主存。主存通常称为随机访问存储器（Random Access Memory, RAM）。

虚拟内存机制方法是将程序放在磁盘上，而将主存作为一种缓存，用来保存最频繁使用的部分程序。这种机制需要快速地映象内存地址，以便把程序生成的地址转换为有关字节在RAM中的物理地址。这种映像由CPU中的一个部件，称为存储器管理单元（Memory Management Unit）来完成。

I/O设备

I/O设备一般包括两个部分：设备控制器和设备本身。专门与控制器对话，发出命令并接收响应的软件，称为设备驱动程序（device driver）。

为了能够使用设备驱动程序，必须把设备驱动程序装入操作系统中，这样它可在核心态中运行。将设备驱动程序装入操作系统，有三个途径：

  • 一是将内核与设备驱动程序重新链接，然后重启动系统。许多UNIX系统以这种方式工作。
  • 二是在一个操作系统文件中设置一个入口，并通知该文件需要一个设备驱动程序，然后重启动系统。在系统启动时，操作系统去寻找所需的设备驱动程序并装载。Windows就是以这种方式工作。
  • 三是操作系统能够在运行时接受新的设备驱动程序并且立即将其安装好，无须重启系统。

实现输入和输出的方式有三种：

  • 一是用户程序发出一个系统调用，内核将其翻译成一个对应设备驱动程序的过程调用，然后设备驱动程序启动I/O
    并在一个连续不断的循环中检查该设备，看该设备是否完成了工作。当I/O
    结束后，设备驱动程序把数据送到指定的地方，并返回。然后操作系统将控制返回给调用者。这种方式成为忙等待（busy
    waiting），其缺点是要占据CPU，CPU一直轮询设备直到对应的I/O操作完成。
  • 二是设备驱动程序启动设备并且让该设备在操作完成时发出一个中断。设备驱动程序在这个时刻返回。
  • 三是为I/O使用一种特殊的直接存储器访问（Direct Memory Access,
    DMA）芯片，它可以控制在内存和某些控制器之间的位流，而无须持续的CPU干预。

管道（pipe）是一种虚文件，它可连接两个进程。

UNIX操作系统对每个文件赋予一个9位的二进制保护代码，对UNIX中的文件实现保护。该保护代码有三个3
位字段，一个用于所有者，一个用于所有者同组中的其他成员，而另一个用于其他人。每个字段中有一个用于读访问，一位用于写访问，一位用于执行访问。这些位就是知名的rwx位。
